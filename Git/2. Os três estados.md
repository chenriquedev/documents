Esta é uma parte bastante importante quando estamos começando aprender o git. O git tem três estados principais onde seus arquivos podem estar: commited, Staging Modified.

- Modified (modificado): Significa que existem alterações em um arquivo no seu Working Directory mas ainda não preparou ele para ser salvo.
- Staging (preparado): Significa que você pegou o arquivo modificado do seu Working Directory  e preparou ele para ser salvo no próximo commit.
- Commited: Significa que seu arquivo está salvo permanentemente no banco de dados do git.

Na imagem acima temos o Working Directory (diretório de trabalho), Staging Area (Área de preparo) e o Repository (o diretório .git).

O **Working Directory** (Diretório de Trabalho) é a sua área de arquivos reais, onde você edita, cria e exclui o código. É onde fica a **única cópia dos arquivos da versão atual** que você está trabalhando.

A **Staging Area** (Área de Preparação) é um **índice**. Ela funciona como um "carrinho de compras" onde você seleciona exatamente **quais alterações** (e não necessariamente o arquivo inteiro) dos seus arquivos modificados serão incluídas no próximo _commit_.

O **.git Directory** (Repositório) é o coração do Git. É onde o Git armazena todo o **histórico de versões** (_commits_), as referências dos seus ramos (_branches_) e todos os metadados do projeto.

O fluxo básico do git é algo parecido com isso:

1. Você modifica os arquivos que estão no Working Directory (diretório de trabalho)
2. Você prepara os arquivos modificados que deseja salvar (Staging Area)
3. Salva as mudanças no banco de dados do git (.git directory)
