O **Git** é um sistema de controle de versão distribuído. Ele serve para o desenvolvedor salvar e rastrear mudanças no código, como um histórico.

### O git salva "imagens"

Durante o desenvolvimento dos nossos projetos usaremos constantemente o git para salvar as alterações feitas no código. Ele funciona como uma máquina do tempo. Toda vez que salvamos o projeto, o git salva uma "imagem" completa do estado do código naquele exato momento e armazena essa imagem no banco de dados interno do git .

### O git trabalha no modo offline

A maioria das operações com o git acontecem no seu computador, sem precisar de internet. A conexão com internet só se torna necessária quando precisamos interagir com repositórios remotos, como o GitHub.

### O git nunca mente (Imutabilidade e Integridade)

Isso quer dizer que é impossível alterar uma versão de arquivos depois que foram salvos no histórico do git. Se um único elemento for alterado, o git vai notar a mudança. O Git faz uma verificação de segurança para garantir a integridade dos arquivos salvos.
