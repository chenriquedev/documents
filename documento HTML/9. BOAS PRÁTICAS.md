#### 1. Use sempre o HTML semântico

Essa é a prática mais importante de todas, não construa o site utilizando apenas a tag `<div>`. O HTML semântico usa tags para dar significado ao conteúdo.

- **O problema:** usar `<div>` e  `<span>` pra tudo . O navegador não consegue identificar o que é o que e isso causa problemas de acessibilidade e SEO.
- **A solução:** Substituir as `<div>` genéricas por tags que descrevem seu propósito:
	- `<header>`: Para o cabeçalho (a tag `<head>` tem outro contexto)
	- `<nav>`: Para o menu de navegação
	- `<main>`: Para o conteúdo principal da página
	- `<aside>`: Para barras laterais 
	- `<section>`: Para agrupar conteúdo tematicamente relacionado (como capítulos, seções de um livro, ou áreas de um site)
	- `<article>`: Para conteúdo independente e auto-contido, que poderia ser distribuído de forma independente (como um post de blog, um artigo de notícia, um comentário).
	- `<footer>`: para o rodapé


Usar essa estrutura melhora a organização e a visibilidade do seu site nos buscadores e facilita na hora de dar manutenção.

#### 2. Metatags essenciais no `<head>` (SEO e Comportamento)

As metatags são elementos invisíveis para o usuário, mas importantes para o navegador e para os mecanismos de busca.

**Meta Charset (``<meta charset="UTF-8">``)**
Define o conjunto de caracteres da página. UTF-8 é o padrão universal e suporta todos os caracteres, incluindo acentos e caracteres especiais.
Sem isso, seu texto pode aparecer com caracteres estranhos.

**Meta Viewport (`<meta name="viewport" content="width=device-width, initial-scale=1.0">`)**
Indica como o site deve se comportar em dispositivos móveis.
Diz para o navegador que a largura da página deve se ajustar à largura do dispositivo.
Essencial para responsividade.

**Meta Description (`<meta name="description" content="Uma breve descrição do seu site ou página para SEO.">`)**
Fornece um resumo do conteúdo da página.
O google e outros buscadores usam esse texto como a descrição que aparecem nos resultados de busca.
#### 3. Conteúdo e Estrutura (SEO e Acessibilidade)

**Usar apenas um `<h1>` por página**
A tag `<h1>` é usada para declarar o título mais importante da página.
Devemos usar as outras tags de *heading* caso necessário.

**Usar o atributo `alt` em todas as imagens**
O atributo `alt` define uma descrição para sua imagem
Caso o usuário utilize leitores de tela ou sua imagem não carregue ela será mostrada.

**Definir o idioma da página com o atributo `lang` na tag `<html>`** 
Isso informa ao navegador o idioma da sua pagina, exemplo: `lang="pt-BR"`
#### 4. Formulários 

**Sempre Usar o elemento `<label>`**
O elemento `<label>` define um rótulo para o input
O valor do atributo `<for>` deve ser o mesmo do atributo `<id>` do input

**Lembrar sempre do atributo `<name>`**
Se o `<input>` não tiver o atributo `<name>` o seu valor não será enviado.

#### 5. Boas práticas Gerais

Lembrar sempre de abrir e fechar as tags corretamente, ou poderá ter erros visuais no site.

Estrutura básica do arquivo HTML sempre vai possuir as tags `<html>` `<head>` `<body>`. Para melhor produtividade, basta usar o ponto de exclamação `(!)` para gerar a estrutura do html
