Os elementos semânticos no HTML são aqueles que tem significado próprio, eles informam ao navegador o tipo de conteúdo que está dentro deles.

>[!TIP]  
Em outras palavras: usar semântica é **dar sentido ao conteúdo**, e não apenas deixá-lo “bonito” com CSS.

Na aula de elementos de bloco foi dito que a tag `<div`>  cria um container para agrupar elementos dentro dela, para posteriormente aplicar estilos, formatações, ajustar o layout e etc.
Mas a tag `<div>` não tem um significado próprio, ele é apenas um container genérico.

**Exemplo da estrutura básica de um site usando DIV**

<div style="width: 100%; text-align: center;"> 
<img src="../imagens/html-nao-semantico.svg" alt="Demonstrando o uso incorreto de html semantico" width="300px">
</div>

Analisando esta imagem, vemos uma estrutura de site clássica. No entanto, ela é construída usando **quase que exclusivamente `<div>`s**.

Isso é um exemplo perfeito de código **não-semântico**. A tag `<div>` é um "divisor" genérico; ela não informa _o que_ ela representa.

Para o navegador, ele não sabe "o que é o quê". Ele não consegue diferenciar o cabeçalho do rodapé, ou um menu lateral do conteúdo principal.

Isso se torna problemático para acessibilidade, otimização (SEO) e manutenção.


**Exemplo da estrutura básica de um site semanticamente correto**

<div style="width: 100%; text-align: center;"> 
<img src="../imagens/html-semantico.svg" alt="Demonstrando o uso incorreto de html semantico" width="300px">
</div>

Agora, observem esta segunda imagem. Esta é a forma **semanticamente correta** de estruturar exatamente a mesma página que vimos antes.

A diferença é fundamental: em vez de `<div>`s genéricas, agora usamos tags de HTML5 que **descrevem o seu propósito** e o _significado_ do seu conteúdo.

- Onde antes era `<div>`, agora é **`<header>`** (cabeçalho).
- Onde era `<span>` (ou outra `<div>`), agora é **`<nav>`** (navegação).
- Onde era a `<div>` da esquerda, agora é **`<aside>`** (barra lateral).
- Onde era a `<div>` principal, agora é **`<main>`** (conteúdo principal).
- Onde era o rodapé, agora é **`<footer>`**.

>[!Tip] Conclusão
>Usar HTML semântico é escrever **código com significado**.  
Isso melhora a **organização**, **acessibilidade** e **visibilidade** do seu site nos buscadores.  
Além disso, torna o código mais **profissional e sustentável**.
